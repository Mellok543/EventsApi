# Events API

ASP.NET Core Web API для управления мероприятиями.

## Требования

* .NET 8 SDK или выше

## Запуск проекта

```bash
dotnet restore
dotnet build
dotnet run
```

После запуска Swagger будет доступен по адресу:

```text
https://localhost:5001/swagger
```

или

```text
http://localhost:5000/swagger
```

Точный адрес будет выведен в консоли после запуска приложения.

## Описание API

### Получить все мероприятия

```http
GET /events
```

Метод поддерживает фильтрацию и пагинацию через query-параметры.

#### Параметры фильтрации

| Параметр | Тип        | Обязательный | Описание                                                              |
| -------- | ---------- | ------------ | --------------------------------------------------------------------- |
| `title`  | `string`   | нет          | Поиск по названию (регистронезависимый, частичное совпадение).        |
| `from`   | `DateTime` | нет          | Вернуть события, которые начинаются не раньше указанной даты (`StartAt >= from`). |
| `to`     | `DateTime` | нет          | Вернуть события, которые заканчиваются не позже указанной даты (`EndAt <= to`).   |

Фильтры применяются только если параметр передан. При указании нескольких
фильтров они работают совместно (логическое И).

#### Параметры пагинации

| Параметр   | Тип   | По умолчанию | Описание                          |
| ---------- | ----- | ------------ | --------------------------------- |
| `page`     | `int` | `1`          | Номер запрашиваемой страницы.     |
| `pageSize` | `int` | `10`         | Количество элементов на странице. |

#### Примеры запросов

```http
GET /events?title=conference
GET /events?from=2026-07-01&to=2026-07-31
GET /events?page=2&pageSize=5
GET /events?title=tech&from=2026-07-01&to=2026-07-31&page=1&pageSize=10
```

Ответ (`200 OK`) возвращается в виде объекта с пагинацией:

```json
{
  "items": [
    {
      "id": "4e6c7b94-2e37-4c59-9c7e-0a4a8d5f7d12",
      "title": "Demo event",
      "description": "Example description",
      "startAt": "2026-06-18T10:00:00",
      "endAt": "2026-06-18T12:00:00"
    }
  ],
  "totalCount": 1,
  "page": 1,
  "pageSize": 10
}
```

* `items` — события на текущей странице;
* `totalCount` — общее количество событий после применения фильтров;
* `page` — номер текущей страницы;
* `pageSize` — количество элементов на странице.

### Получить мероприятие по ID

```http
GET /events/{id}
```

Если мероприятие найдено, возвращается `200 OK`.

Если мероприятие не найдено, возвращается `404 Not Found`.

### Создать мероприятие

```http
POST /events
```

Тело запроса:

```json
{
  "title": "Demo event",
  "description": "Example description",
  "startAt": "2026-06-18T10:00:00",
  "endAt": "2026-06-18T12:00:00"
}
```

При успешном создании возвращается `201 Created`.

### Обновить мероприятие

```http
PUT /events/{id}
```

Тело запроса:

```json
{
  "title": "Updated event",
  "description": "Updated description",
  "startAt": "2026-06-18T13:00:00",
  "endAt": "2026-06-18T15:00:00"
}
```

Если мероприятие найдено, возвращается `200 OK`.

Если мероприятие не найдено, возвращается `404 Not Found`.

### Удалить мероприятие

```http
DELETE /events/{id}
```

Если мероприятие найдено и удалено, возвращается `204 No Content`.

Если мероприятие не найдено, возвращается `404 Not Found`.

## Валидация

Поля `title`, `startAt`, `endAt` обязательны.

Поле `endAt` должно быть позже `startAt`.

Если данные не проходят валидацию, API возвращает `400 Bad Request`.

## Обработка ошибок

Все необработанные исключения перехватываются глобальным middleware
(`ExceptionHandlingMiddleware`), который возвращает единообразный JSON-ответ в
формате Problem Details (RFC 7807).

| Ситуация                                   | HTTP-статус                 |
| ------------------------------------------ | --------------------------- |
| Ошибка валидации бизнес-логики             | `400 Bad Request`           |
| Ресурс не найден                           | `404 Not Found`             |
| Непредвиденная ошибка                      | `500 Internal Server Error` |

Пример ответа при отсутствии ресурса (`404 Not Found`):

```json
{
  "title": "Not Found",
  "status": 404,
  "detail": "Event with id '11111111-2222-3333-4444-555555555555' was not found",
  "instance": "/events/11111111-2222-3333-4444-555555555555"
}
```

Пример ответа при ошибке валидации (`400 Bad Request`):

```json
{
  "title": "Bad Request",
  "status": 400,
  "detail": "Page must be greater than or equal to 1",
  "instance": "/events"
}
```

Ошибки уровня непредвиденных исключений (`500`) логируются через встроенный
`ILogger`, а клиенту возвращается обобщённое сообщение без деталей реализации.

## Тестирование

Юнит-тесты бизнес-логики сервиса `EventService` находятся в проекте
`EventsApi.Tests` и написаны с использованием фреймворка **xUnit**.

Тесты покрывают:

* успешные CRUD-сценарии (создание, получение всех, получение по ID, обновление,
  удаление);
* фильтрацию по названию и датам, пагинацию и комбинированную фильтрацию;
* неуспешные сценарии (несуществующий ID, некорректные данные, `EndAt` раньше
  `StartAt`, некорректные параметры пагинации).

Запуск тестов из корня репозитория:

```bash
dotnet test
```
